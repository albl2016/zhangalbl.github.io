构造函数的继承

发表于 2016-07-15   |  

在面向对象中，构造函数是一种特殊的方法，我们可以使用new关键字实例化对象，构造函数可以为对象的成员变量赋初始值，在javascript中，构造函数必须使用new关键字来实例化对象，例如：

构造函数属性的继承

定义了一个Person的构造函数：

function Person(name,age){
    this.name = name;
    this.age = age;
}

Person.prototype.say = function(){
    console.log("我是Person的方法");
}

var person1 = new Person("小明",19);
var person2 = new Person("小红",18);
console.log(person1);
console.log(person2);

上面的代码实例化两个对象person1和person2，他们有共同的方法，即say();

如果我们要创建一个儿童对象的构造函数，我们可以继承这个Person的一些属性的方法（使用call和apply方法）如:

function Child(name,age,gender){
    //name和age是从Person构造函数继承的属性，下面call和apply方法去继
承这两个属性
    Person.call(this,name,age);
    Person.apply(this,[name,age]);    
    Person.apply(this,arguments);

    this.gender = gender;        //gender是Child对象自己的属性
}
var child1 = new Child("xiaowang",19,"男");
console.log(child1);

在Child()构造函数继承了Person()的name,age属性，使用了call和apply两种方法：

call的语法是：call([thisObj[,arg1[, arg2[, [,.argN]]]]])；

call 方法可将一个函数的对象上下文从初始的上下文改变为由 thisObj 指定的新对象。如果没有提供thisObj 参数，那么 Global 对象被用作thisObj。说明白一点其实就是更改对象的内部指针，即改变对象的this指向的内容。在面向对象的js编程过程中有时是很有用。

apply语法：apply([thisObj[,argArray]])

如果 argArray 不是一个有效的数组或者不是 arguments 对象，会提示错误TypeError。如果没有提供 argArray 和 thisObj 任何一个参数，那么 Global 对象将被用作 thisObj，并且无法被传递任何参数。

构造函数原型的继承

function Person(name,age){
    this.name = name;
    this.age = age;
}    
Person.prototype.move = function(){
    console.log("我是Person");
}

function Student(name,age,gender){
    this.gender = gender;
    Person.call(this,name,age);
}
//Student.prototype = Person.prototype;    //原型继承这是不正确的，前面说过传址的问题，如果子类(Student)修改了子类原型的方法会影响父类(Person)原型的方法。
function Lin(){};            //创建空的构造函数,
Lin.prototype = Person.prototype;    //通过赋值实现传址，Lin原型上的内容和Person相同
var pro = new Lin();    //实例化Lin，使其具有自己的地址空间，和之前的Person地址不同
Student.prototype = pro;    //用变量接收实例化出来的对象，赋值给Student.prototype这时候Student的地址和Person的地址不同
Student.prototype.constructor = Student;   // 将原型的指向改回Student,完美

因为每个prototype对象和它的实例都有一个constructor属性，指向它的构造函数，所以在这里，Student和Lin的constructor属性都指向Person，会造成继承链的紊乱，所以，我们使用

Student.prototype.constructor = Student;

将Student.prototype.constructor的值改为Student。
