CSS中px,em,rem三种单位之间的区别

Veröffentlicht am 2016-06-23

px像素（pixel）单位

像素px单位是css中常见的长度单位，可以很精确地展示页面内容的大小，但是当用户放大或缩小页面时可能使页面内容产生变形，移位。

em单位

em是一种相对单位，它是相对于它的父级元素的字号的大小来设置，比较适合一些自适应屏幕尺寸的终端布局方式.一般来说，浏览器默认的字体大小是16px,也就是说1em等于16px,我们要给别的文本设置字体大小时，就需要用公式：em = 需要设置字体的大小/16px，得出以em为单位的尺寸。当然，在一个页面内，1em并不是固定的大小，它会随着父级元素大小改变，例如：我们在body中用一个h2标签嵌套一个span标签，在css中输入下方的代码：


body{
    margin: 0;
    font-size: 12px;   /设置body中字体大小为12px/
}
h2{
    font-size: 2em;       /设置h2中的文本字体大小为2em就是212px=24px;因为它的父级是body属性 /
}
h2 span{
    font-size: 1.5em;   /设置span中文本大小为1.5em就是1.524px=36px;它的父级是h2，所以它的大小是相对它的父级而改变的/
}

如果我们的css文件都采用em作为单位计算大小的话，那么在屏幕尺寸的终端上浏览我们的页面时，不容易发生页面错乱的问题。

rem单位

rem和em一样是一个相对单位，我们可以理解为root em，就是相对于根元素计算的单位，如果我们为根元素设置有一个固定值，那么整个页面的1rem都相等，计算方式和em是一样的，我们只要改变根元素的大小就能成比例的调整整个页面的尺寸，例如：


html{
    font-size: 16px;   /设置html根元素字体大小为16px/
}
body{
    margin: 0;
}
h2{
    font-size: 2rem;   /设置h2中的文本字体大小为2rem就是216px=32px/
}
h2 span{
    font-size: 2rem;   /设置span中文本大小为1.5rem就是216px=32px;/
}

hexo博客的搭建

Veröffentlicht am 2016-06-22

经过一天的学习，在韩老师的指导下完成了自己hexo博客的搭建，下面和大家分享一下搭建的一些步骤，有什么不对的地方欢迎指正，废话不多说，直接上步骤。

首先是配置的环境

安装Node，用来生成静态的页面。在这里我们可以直接去Node.js的官方网站下载安装包，安装即可,安装完成后输入 node -v测试一下。

安装hexo

Node安装完成后，我们打开终端执行以下命令安装Hexo:

1.sudo npm install hexo -g

输入管理员密码，然后输入hexo测试一下。

2.在桌面（其他位置也可）创建一个blog文件夹。使用cd命令进入这个文件夹，在该目录下，输入命令：hexo init 初始化hexo目录，输入命令：hexo server 来启动本地服务（按 Ctrl+C 两次可以结束本地服务）。如下图：

在浏览器地址栏中输入“localhost:4000”，可以打开对应的本地的静态网页，就是hexo博客首页，这个blog文件夹就是我们的博客根目录。

配置github

1.先去github官网注册github账号,注册成功后打开终端，输入命令：ssh-keygen 连续按三次回车键，接着输入命令：vim ~/.ssh/id_rsa.pub 会出现一串字母如图：

把这串SSH keys复制下来粘贴到你注册github账号设置的界面下Settings菜单中SSH and GPG keys子菜单中.

2.接下来配置github和本地机器验证，在终端中输入命令：git config –global user.name “你的用户名”然后输入命令：git config –global user.email “你注册账号的邮箱地址”

3.在github中创建域名仓库，注意格式：你的用户名.github.io 这就是你的域名http地址。

4.打开blog文件夹中“_config.yml”文件，在文件内容最下方的type：后面加上git（注意git前面有个空格）在下面继续添加一项repo： 域名仓库的地址（注意前面有个空格），如下图：

将静态页面部署到github上

在终端中输入命令：npm install hexo-deployer-git –save 完成后输入命令：hexo g 再输入命令hexo d，将静态网页部署到github上就完成了。
