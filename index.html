Meta标签的作用

申明编码

meta提供关于HTML文档的元数据，元数据不会显示在页面上，但是对于计算机来说是可读的。它可用于浏览器（如何显示内容或者重新加载页面），搜索引擎（关键词），或者其他web服务。

<meta charset="utf-8"/>  标签是一个单标签，在html文档中用来申明编码格式。

浏览器内核控制

我们国内的浏览器很多都是双内核（webkit和trident）,webkit内核高速浏览，IE内核兼容网页和旧版网站。而添加meta标签的网站可以控制浏览器选择何种内核渲染。例如我们编写如下代码：

<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1"/>

那么会优先使用IE最新版本和chrome的内核来渲染页面。

国内双核浏览器默认内核模式如下：

1、搜狗浏览器、QQ浏览器：IE内核（兼容模式）

2、360极速、遨游浏览器：Webkit内核（极速模式）

<meta name="renderer" content="webkit|ie-comp|ie-stand"/>

页面关键字

用来说明该网页内容的一组唯一的关键字，人们可能会搜索，并准确描述网页上所提供信息的描述性和代表性关键字及短语。（一般关键字不超过15个）

<meta name="keywords" content="your tags"/>

页面描述

每个网页都应有能准确反映网页内容的描述标签(字符数在120~150之间最好，不超过150个字符)。

<meta name="description"content="120｜150 words"/>

页面重定向和刷新

<meta http-equiv="refresh" content="0;url="/>

content内的数字代表时间（秒），即多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

搜索引擎索引方式

搜索引擎索引方式，robotterms是一组使用逗号(,)分割的值，通常有如下几种取值：none,noindex,nofollow，all，index和follow。确保正确使用nofollow和noindex属性值。

<meta name=“robots” content="index,follow"/>

all：文件将被检索，且页面上的链接可以被查询；

none：文件将不被检索，且页面上的链接不可以被查询；

index：文件将被检索；

follow：页面上的链接可以被查询；

noindex：文件将不被检索；

nofollow：页面上的链接不可以被查询。

其他的作用

1、网页作者信息

<meta name="author" content="author name"/>

2、移动设备自适应屏幕（对PC端没有影响，对移动端显示更好，非常适合在移动端使用）

<meta name="viewport" content="width=device-width, initial-scale=1.0,maximum-scale=1.0, user-scalable=no"/>

3、WebApp全屏模式：伪装app，离线应用

<meta name="apple-mobile-web-app-capable" content="yes"/>

4、隐藏状态栏/设置状态栏颜色：只有在开启WebApp全屏模式时才生效

<meta name="apple-mobile-web-app-status-bar-style"content="black-translucent"/>

content的值为 default|black|black-translucent

5、添加到主屏后的标题

<meta name="apple-mobile-web-app-title"content="标题"/>

content的值为default|black|black-translucent

6、忽略数字自动识别为电话号码

<meta content="telephone=no"name="format-detection"/>

7、忽略识别邮箱

<meta content="email=no"name="format-detection"/>

发表于 2016-06-28   |  

子元素选择器 vs后代选择器

格式：E > F VS E F

子元素选择器选择的是E的子元素F,不包括F的后代元素，而后代选择器选择的是E后代F，包括子子孙孙。

    .div1 > h2{            /* 表示选择div1的子元素的h2,对其设置一个1像素的红色边框 */
        border: 2px red solid;
    }
    .div1 h2{               /* 表示选择div1下面所有的后代子孙的h2,设置背景为红色*/
        background: red;
    }

兄弟选择器

分为：相邻兄弟选择器 vs 通用兄弟选择器

格式： E + F VS E ~ F

相邻兄弟选择器（父级相同）选择的必须是同级的、相邻的（重点是相邻元素）、在它下方（不是它的子元素，是同级的从上到下排列，位置在上方也不行）的元素。例如：

    div + h2{
        border: 1px blue solid;            /* 选择div下方和它自己相邻而且同级的h2，对不相邻的不会选择，只有一个*/
    }

    div ~ h2{
        border: 2px red solid;            /* 选择div下方和它同级的的元素h2，对不相邻的也会选择，可以选择多个 */
    }

属性选择器

格式：E[attr] 例如：E[class]表示选择属性名为class的元素E

E[attr=“value”] 例如：E[class=”div1”]表示选择属性名为class且属性值为”div1”的元素E

E[attr~=“value”] 例如：E[class~=”div1”]表示选择几个以空格隔开的属性值中包含了”div1”的元素E

E[attr ^ =“value”] 例如：E[class^=”div1”]表示选择以属性值为”div1”开头的元素E

E[attr $ =“value”] 例如：E[class$=”div1”]表示选择以属性值为”div1”结尾的元素E
E[attr =“value”] 例如：E[class=”div1”]表示选择属性值包含了”div1”的元素E

E[attr|=“value”] 例如：E[class|=”div1”]表示选择以属性值为”div1”或”div1-“开头的元素E

例如我们要实现将超链接变成红色，可以这样写:

a[href="http://www.baidu.com"] {color: red;}

伪类选择器

伪类选择器主要分为锚点伪类和UI元素伪类（针对表单元素）。

锚点伪类主要是link，visited，hover，active和focus.它们表达的意思是：

:link 元素没有被访问的状态。

:visited 元素被访问过后的状态。

:hover 鼠标悬浮在元素上的状态。

:active 点击元素那一下的状态。

:focus 表示元素成为焦点的状态。

例如： input:focus{border:10px red solid;} 运行结果如下图：

在input元素中，type=”text”有两种不同的状态分别是disabled(不可编辑状态)和enabled(可编辑状态).type=”checkbox”也有checked和unchecked两种状态。例如：

        input[type="text"]:disabled{border: 4px red solid;}

CSS中px,em,rem三种单位之间的区别

发表于 2016-06-23   |  

px像素（pixel）单位

像素px单位是css中常见的长度单位，可以很精确地展示页面内容的大小，但是当用户放大或缩小页面时可能使页面内容产生变形，移位。

em单位

em是一种相对单位，它是相对于它的父级元素的字号的大小来设置，比较适合一些自适应屏幕尺寸的终端布局方式.一般来说，浏览器默认的字体大小是16px,也就是说1em等于16px,我们要给别的文本设置字体大小时，就需要用公式：em = 需要设置字体的大小/16px，得出以em为单位的尺寸。当然，在一个页面内，1em并不是固定的大小，它会随着父级元素大小改变，例如：我们在body中用一个h2标签嵌套一个span标签，在css中输入下方的代码：

body{
    margin: 0;
    font-size: 12px;    /*设置body中字体大小为12px*/
}
h2{
    font-size: 2em;       /*设置h2中的文本字体大小为2em就是2*12px=24px;因为它的父级是body属性 */
}
h2 span{
    font-size: 1.5em;   /*设置span中文本大小为1.5em就是1.5*24px=36px;它的父级是h2，所以它的大小是相对它的父级而改变的*/
}

如果我们的css文件都采用em作为单位计算大小的话，那么在屏幕尺寸的终端上浏览我们的页面时，不容易发生页面错乱的问题。

rem单位

rem和em一样是一个相对单位，我们可以理解为root em，就是相对于根元素计算的单位，如果我们为根元素设置有一个固定值，那么整个页面的1rem都相等，计算方式和em是一样的，我们只要改变根元素的大小就能成比例的调整整个页面的尺寸，例如：


html{
    font-size: 16px;   /设置html根元素字体大小为16px/
}
body{
    margin: 0;
}
h2{
    font-size: 2rem;   /设置h2中的文本字体大小为2rem就是216px=32px/
}
h2 span{
    font-size: 2rem;   /设置span中文本大小为1.5rem就是216px=32px;/
}

1.先去github官网注册github账号,注册成功后打开终端，输入命令：ssh-keygen 连续按三次回车键，接着输入命令：vim ~/.ssh/id_rsa.pub 会出现一串字母如图：

把这串SSH keys复制下来粘贴到你注册github账号设置的界面下Settings菜单中SSH and GPG keys子菜单中.

2.接下来配置github和本地机器验证，在终端中输入命令：git config –global user.name “你的用户名”然后输入命令：git config –global user.email “你注册账号的邮箱地址”

3.在github中创建域名仓库，注意格式：你的用户名.github.io 这就是你的域名http地址。

4.打开blog文件夹中“_config.yml”文件，在文件内容最下方的type：后面加上git（注意git前面有个空格）在下面继续添加一项repo： 域名仓库的地址（注意前面有个空格），如下图：

将静态页面部署到github上

在终端中输入命令：npm install hexo-deployer-git –save 完成后输入命令：hexo g 再输入命令hexo d，将静态网页部署到github上就完成了。
